Infrastructure & Foundation
Week 1 – 2 • Days 1–10
Week 1 — Project Setup & Database
Day 1-2 Environment & Meta App Process
  • Initialize Git repo, Docker Compose (PostgreSQL, Redis, FastAPI, Celery)
  • Python venv, requirements.txt, .env.example
  • Create full project directory structure
  • START META BUSINESS VERIFICATION (critical path)
  • Register Reddit API app, verify Leonardo.ai key, set up Cloudflare R2
Day 3-4 Database Schema
  • SQLAlchemy models: trending_content, bible_verses, generated_content, generated_images, posting_log, analytics_snapshots, competitor_snapshots, content_calendar, affiliate_links, affiliate_clicks, email_subscribers, revenue_log, brand_contacts, hashtag_performance
  • Alembic initial migration + seed data script
Day 5 FastAPI Skeleton
  • FastAPI app with CORS, health check, route modules
  • Celery app + Beat schedule config
  • API key auth middleware
  • Docker Compose full stack verification
Week 2 — Core API Clients
Day 6-7 Bible API + Reddit Scraper
  • bible_api.py: fetch by reference, random verse rotation, 90-day dedup, caching
  • reddit_scraper.py: PRAW client, 6 subreddits, engagement scoring, rate limiting
  • Tests + Celery tasks
Day 8-9 Google Trends + Pinterest
  • google_trends.py: pytrends with faith/family keyword groups, scoring
  • pinterest_scraper.py: web scraping trending pins, engagement signals
  • Tests for both
Day 10 Competitor Tracker + Hashtag Research + Aggregator
  • competitor_tracker.py: 5 target pages, public data, engagement rate calc
  • hashtag_research.py: 90 curated hashtags (30 per tier), scoring
  • trend_aggregator.py: normalize scores 0-100, composite ranking, daily brief output
2
Content Generation Engine
Week 3 – 4 • Days 11–20
Week 3 — Prompt Templates & Claude Integration
Day 11-12 Prompt Template System
  • All prompt files: system_prompt, daily_verse, marriage_monday, parenting_wednesday, faith_friday, viral_formats, devotional_book, image_prompt_guidelines
  • prompt_templates.py: Jinja2 loading, variable injection, validation, versioning
Day 13-14 Claude API Content Generator
  • generator.py: Anthropic SDK, system prompt loading, per-type generation
  • JSON response parsing + validation (15+ keys, hashtag counts, char limits)
  • Retry logic, cost tracking, rate limiting, batch generation
  • Model: claude-sonnet-4-6
Day 15 Content Calendar Logic
  • calendar_logic.py: Weekly slot allocation (Mon=Marriage, Tue=Verse, Wed=Parenting, Thu=Viral, Fri=Hardship, Sat=Reel, Sun=Gratitude)
  • Posting times: 6:30am, 12pm, 7:30pm EST
  • Diversity enforcement, tone rotation, 7-day lookahead, JSON export
Week 4 — Pipeline Integration
Day 16-17 Series Manager
  • series_manager.py: theme libraries for each series, age group rotation, 8-week no-repeat rule, trending topic integration
Day 18-19 End-to-End Content Pipeline
  • Wire: trends → calendar → series → generator → DB (status=pending)
  • Celery orchestration, regeneration endpoint, duplication detection
  • Full pipeline test with real API calls
Day 20 Quality Assurance
  • All templates produce valid JSON, calendar produces balanced weeks
  • Hashtags properly tiered, image prompts follow guidelines
  • 7-day pipeline simulation

Image Generation Pipeline
Week 5 – 6 • Days 21–28
Week 5 — Leonardo.ai + Fallback
Day 21-22 Leonardo.ai Client
  • leonardo_client.py: REST API, model selection (Diffusion XL / AlbedoBase XL / Creative)
  • Params: guidance_scale=7, steps=30, aspect ratio handling (4:5, 9:16, 1:1)
  • Async polling, download, quota tracking, retry logic
Day 23-24 Unsplash + Image Processing
  • unsplash_client.py: search, keyword mapping, attribution, rate limits
  • image_processor.py: Pillow resize/crop, optimization, format conversion, R2 upload
Week 6 — Canva + Brand System
Day 25-26 Canva Client
  • canva_client.py: OAuth2, template management (scripture, encouragement, carousel, story)
  • Brand: Playfair Display + Lato fonts, cream/gold/green/white palette
  • Dynamic text injection, multi-format export, template rotation
Day 27-28 PIL Fallback + Pipeline Integration
  • PIL-based text overlay backup (brand fonts, shadows, blur)
  • Full image pipeline: prompt → Leonardo → Canva → optimize → R2 → DB
  • Test with 10 sample pieces

Day 39-40 Email Notifications + E2E Testing
  • SendGrid: success/failure alerts, daily summary, weekly digest
  • Full E2E: generate → image → approve → post → verify → log
