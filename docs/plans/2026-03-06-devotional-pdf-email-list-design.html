Design: Devotional PDF Generator + Email List Setup

Date: 2026-03-06 | Approved

Overview

Two-phase feature: (1) a templated devotional PDF generator producing branded 7-day devotionals as lead magnets, and (2) ConvertKit email list integration for subscriber growth with automated welcome sequences.

Phase 1: Devotional PDF Generator

Product

"Finding Peace in Every Season" — a 7-day branded devotional PDF (16-18 pages).

Page Structure

PageContent
1Cover — hero Unsplash nature image, title in Georgia, @stillwatergrace branding
2Welcome — brief intro, how to use this devotional, about Still Water Grace
3-16Day 1-7 (2 pages each) — image page + reflection page
17Closing — encouragement, CTA to follow on Instagram, share with a friend
18About / Connect — social links, website, brief mission statement

Each Day (2-page spread)

  • Page A (Image): Day number + theme title, full-bleed Unsplash photo with verse overlay (styled like Instagram cards), verse reference
  • Page B (Reflection): Devotional reflection (200-250 words, Claude-generated), "Reflect" question (1-2 prompts), written prayer (3-4 sentences), journaling lines

Visual Design

  • Cream #FFF8F0 page background with subtle paper texture
  • Gold #D4A853 accent lines, dividers, day numbers
  • Green #2D4A3E headings and verse references
  • Georgia for headings/verses, Calibri for body text
  • Watercolor wash elements on section transitions
  • @stillwatergrace watermark in footer

Content Generation (Hybrid Approach)

  • Bible verses: pulled from existing DB cache (curated list per theme)
  • Devotional text: Claude API generates reflections, prayers, and reflection questions
  • Images: Unsplash (1 cover + 7 day images, mood-matched to verse content)
  • Verse overlay cards: reuse existing PIL image_processor.py

Technical Stack

  • WeasyPrint — HTML/CSS to PDF rendering
  • Jinja2 — reusable page templates (cover, day_a, day_b, closing)
  • Claude API — devotional reflection generation
  • Unsplash — nature/landscape photography
  • PIL — verse overlay cards (existing code)
  • R2 — final PDF upload for public download URL

Templated System

Theme configs define: title, subtitle, 7 verse references, mood keywords for Unsplash queries. Swapping theme config produces a new devotional with no code changes.

CLI Commands

python manage.py generate-devotional --theme "Finding Peace in Every Season"
python manage.py generate-devotional --theme "Strength for the Family"
python manage.py list-devotionals

Phase 2: Email List (ConvertKit)

Architecture Split

  • ConvertKit = audience growth (signup, lead magnet delivery, welcome sequence, weekly devotional email)
  • SendGrid = system ops (posting failure alerts, weekly analytics digest) — already in config

ConvertKit Setup

  1. Create free ConvertKit account (free up to 10,000 subscribers)
  2. Create landing page: "Get Your Free 7-Day Devotional"
  3. Upload PDF as incentive email attachment (auto-delivered on signup)
  4. Build welcome email sequence

Welcome Sequence (5 emails, 14 days)

EmailTimingContent
1ImmediateWelcome + PDF download link
2Day 3"How's Day 1 going?" + encouragement verse
3Day 5Best-of content from Instagram (curated)
4Day 10Personal story / testimony
5Day 14"Stay connected" — follow on socials, share with a friend

Integration (Minimal Code)

  • Add convertkit_api_secret to config
  • New core/email/convertkit_client.py — thin wrapper to pull subscriber count
  • Dashboard widget showing subscriber count
  • Instagram bio link points to ConvertKit landing page URL
  • Optional: Celery task to sync subscriber count daily for analytics

What ConvertKit Handles (No Code Needed)

  • Landing page design and hosting
  • Email sequence automation
  • Subscriber management, tagging, unsubscribe
  • PDF delivery to new subscribers
  • GDPR/CAN-SPAM compliance

Out of Scope (YAGNI)

  • No custom landing page (ConvertKit provides one)
  • No subscriber database table (ConvertKit manages this)
  • No email sending logic (ConvertKit handles delivery)
  • No unsubscribe handling (ConvertKit handles compliance)
  • No A/B testing (premature at this stage)
  • No personalized PDFs per subscriber
