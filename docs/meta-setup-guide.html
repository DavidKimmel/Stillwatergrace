Meta Platform Setup Guide (Instagram + Facebook)

Step-by-step guide for connecting a new niche page to the StillWaterGrace platform. Covers Meta Developer App, Instagram API, Facebook Page API, and token generation.

Prerequisites

  • A Facebook account that owns/admins the page
  • A Facebook Page (New Pages Experience is fine)
  • An Instagram Professional account linked to the Facebook Page
  • A Meta Business Portfolio containing the page

Critical Gotchas (Read First!)

1. Facebook Page IDs — TWO different IDs exist:
New Pages Experience pages have a Profile ID (visible in the URL, e.g., facebook.com/profile.php?id=61552254632726) and an API Page ID (found in Business Manager > Settings > Pages, e.g., 1071328066057500).

These are completely different numbers. The profile ID will return "object does not exist" errors from the Graph API. Always use the API Page ID from Business Manager.
2. Business Portfolio requirement:
The page MUST be in a Meta Business Portfolio for the API to see it. Without this, me/accounts returns empty even with correct permissions. Go to business.facebook.com and add the page to a Business Portfolio under Settings > Accounts > Pages.
3. Token + Page selection:
When generating tokens in Graph API Explorer, the Facebook authorization popup has a step to select which Pages to grant access to. Don't click through it quickly — make sure your page is checked.
4. me/accounts returns empty — this almost always means:
  • Wrong token (missing pages_show_list permission)
  • Page not in a Business Portfolio
  • Page not selected during the authorization popup flow

Step 1: Create the Meta Developer App

  1. Go to developers.facebook.com/apps
  2. Click "Create App" > Business type
  3. Name it (e.g., "Still Water Grace")
  4. Link it to your Business Portfolio

Step 2: Add Use Cases & Permissions

Instagram API (for posting to Instagram)

This should be added by default. Customize the use case and ensure these permissions are added:

  • instagram_basic
  • instagram_content_publish
  • instagram_manage_comments
  • pages_show_list
  • pages_read_engagement

Manage Everything on Your Page (for Facebook posting)

  1. From Dashboard, click "Add use cases"
  2. Filter by "Content management"
  3. Select "Manage everything on your Page"
  4. Click Save
  5. Then customize that use case and click + Add on:
    • pages_manage_posts — required for publishing to the page

Step 3: Find the Correct Facebook Page ID

THIS IS THE #1 SETUP MISTAKE. Do not skip this.
  1. Go to business.facebook.com
  2. Select your Business Portfolio
  3. Go to Settings (gear icon) > Accounts > Pages
  4. Click your page
  5. The ID number shown next to the page name is the correct API Page ID
  6. It will be different from the number in your facebook.com/profile.php URL
Example:
URL shows: facebook.com/profile.php?id=61552254632726 ← WRONG for API
Business Manager shows: ID: 1071328066057500 ← CORRECT for API

Step 4: Find the Instagram Business Account ID

  1. In the Graph API Explorer
  2. Select your app, User Token
  3. Query: GET /me/accounts?fields=id,name,instagram_business_account
  4. Find your page in the results
  5. The instagram_business_account.id is your INSTAGRAM_BUSINESS_ACCOUNT_ID

Step 5: Generate Access Token

  1. Go to Graph API Explorer
  2. Select your app under "Meta App"
  3. Select "User Token" under "User or Page"
  4. Add all permissions from Step 2 in the Permissions section
  5. Click "Generate Access Token"
  6. In the popup: make sure to select your Page when asked
  7. Copy the short-lived token

Step 6: Exchange for Long-Lived Token

Short-lived tokens expire in ~1 hour. Exchange it:

curl "https://graph.facebook.com/v19.0/oauth/access_token?\
grant_type=fb_exchange_token&\
client_id=YOUR_APP_ID&\
client_secret=YOUR_APP_SECRET&\
fb_exchange_token=YOUR_SHORT_LIVED_TOKEN"

The response contains a long-lived token (60 days or non-expiring).

Or use the CLI if you already have a valid token: python manage.py token-status --refresh

Step 7: Update .env

META_APP_ID=your_app_id
META_APP_SECRET=your_app_secret
INSTAGRAM_ACCESS_TOKEN=your_long_lived_token
INSTAGRAM_BUSINESS_ACCOUNT_ID=your_ig_business_account_id
FACEBOOK_PAGE_ID=your_correct_api_page_id  # FROM BUSINESS MANAGER, NOT THE URL!

Step 8: Verify

# Check token health
docker compose exec api python manage.py token-status

# Test Facebook connectivity
docker compose exec api python -c "
from core.posting.facebook_client import FacebookClient
client = FacebookClient()
print('Page token:', 'OK' if client.page_token != client.user_token else 'FAILED')
"

# Test Instagram connectivity
docker compose exec api python -c "
from core.posting.instagram_client import InstagramClient
client = InstagramClient()
print('Instagram client: OK')
"

Token Auto-Refresh

  • Celery beat task runs every Wednesday at 3 AM EST
  • Refreshes if token has < 14 days remaining
  • Updates .env file automatically
  • Manual check: python manage.py token-status
  • Manual refresh: python manage.py token-status --refresh
