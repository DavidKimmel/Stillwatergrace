StillWaterGrace - Daily Workflow Guide

Prerequisites

  • Docker Desktop must be installed and running
  • Project folder: C:\stillwatergrace
  • .env file configured with API keys

Startup (Once Per Session)

  1. Open Docker Desktop (just let it boot, no clicks needed)
  2. Open a terminal in C:\stillwatergrace
  3. Run: 
    docker compose up -d
  4. Open your dashboard: http://localhost:5175

What Runs Automatically

When Docker is running, these services handle everything:

ServiceWhat It DoesPort
dbPostgreSQL database (stores all content, images, logs)5432
redisMessage broker for Celery task queue6379
apiFastAPI backend (REST API for dashboard)8000
celery-workerExecutes tasks (content generation, image processing, posting)-
celery-beatScheduler (triggers daily generation + posting at set times)-
dashboardReact web UI for reviewing and managing content5175

Daily Flow

  1. Celery Beat triggers content generation each morning
  2. Celery Worker calls Claude API to generate content, creates images, uploads to R2
  3. Content appears in dashboard as "pending"
  4. You review and approve (or auto-approve is enabled)
  5. At scheduled posting times (6:30 AM, 12:00 PM, 7:30 PM EST), the posting task picks up approved content and publishes to Instagram

Content Approval

Two modes:

  • Manual (default): Content generates as "pending". Review in dashboard, approve what you like.
  • Auto-approve: Add AUTO_APPROVE_CONTENT=true to your .env file. All generated content is immediately approved and will post at its scheduled time.
Important: After changing .env, you must restart the affected services: 
docker compose down api celery-worker celery-beat
docker compose up -d api celery-worker celery-beat
Note: docker compose restart does NOT reload .env changes.

Common Commands

# Start everything
docker compose up -d

# Stop everything
docker compose down

# Stop and wipe database (fresh start)
docker compose down -v

# View logs
docker compose logs api -f          # API logs
docker compose logs celery-worker -f # Worker logs
docker compose logs celery-beat -f   # Scheduler logs

# Check service status
docker compose ps

# Manually trigger content generation
docker compose exec api python manage.py generate-content

# View the content calendar
docker compose exec api python manage.py show-calendar

# Run tests
docker compose exec api python -m pytest tests/

Troubleshooting

Docker Desktop must stay running

If you close Docker Desktop, all services stop. No content generates, no posts go out. Reopen Docker Desktop and run docker compose up -d to restart.

Container won't start?

docker compose down
docker compose up -d --build

The --build flag rebuilds containers if code changed.

.env changes not taking effect?

Must do docker compose down [service] && docker compose up -d [service]. Plain restart does not reload environment variables.
